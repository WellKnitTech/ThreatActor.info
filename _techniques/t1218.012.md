## Description

Adversaries may abuse verclsid.exe to proxy execution of malicious code. Verclsid.exe is known as the Extension CLSID Verification Host and is responsible for verifying each shell extension before they are used by Windows Explorer or the Windows Shell. [WinOSBite verclsid.exe](https://winosbite.com/verclsid-exe/)

Adversaries may abuse verclsid.exe to execute malicious payloads. This may be achieved by running <code>verclsid.exe /S /C {CLSID}</code>, where the file is referenced by a Class ID (CLSID), a unique identification number used to identify COM objects. COM payloads executed by verclsid.exe may be able to perform various malicious actions, such as loading and executing COM scriptlets (SCT) from remote servers (similar to Regsvr32). Since the binary may be signed and/or native on Windows systems, proxying execution via verclsid.exe may bypass application control solutions that do not account for its potential abuse. [LOLBAS Verclsid](https://lolbas-project.github.io/lolbas/Binaries/Verclsid/) [Red Canary Verclsid.exe](https://redcanary.com/blog/verclsid-exe-threat-detection/) [BOHOPS Abusing the COM Registry](https://bohops.com/2018/08/18/abusing-the-com-registry-structure-part-2-loading-techniques-for-evasion-and-persistence/) [Nick Tyrer GitHub](https://gist.github.com/NickTyrer/0598b60112eaafe6d07789f7964290d5)
