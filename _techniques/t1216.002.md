## Description

Adversaries may abuse SyncAppvPublishingServer.vbs to proxy execution of malicious PowerShell commands. SyncAppvPublishingServer.vbs is a Visual Basic script associated with how Windows virtualizes applications (Microsoft Application Virtualization, or App-V).[1 - appv](https://securelist.com/bluenoroff-methods-bypass-motw/108383/) For example, Windows may render Win32 applications to users as virtual applications, allowing users to launch and interact with them as if they were installed locally.[2 - appv](https://learn.microsoft.com/en-us/windows/application-management/app-v/appv-getting-started)[3 - appv](https://www.hackingarticles.in/indirect-command-execution-defense-evasion-t1202/)
    
The SyncAppvPublishingServer.vbs script is legitimate, may be signed by Microsoft, and is commonly executed from `\System32` through the command line via `wscript.exe`.[4 - appv](https://www.trellix.com/en-ca/about/newsroom/stories/research/suspected-darkhotel-apt-activity-update/)[5 - appv](https://lolbas-project.github.io/lolbas/Scripts/Syncappvpublishingserver/)

Adversaries may abuse SyncAppvPublishingServer.vbs to bypass PowerShell execution restrictions and evade defensive counter measures by "living off the land."[6 - appv](https://strontic.github.io/xcyclopedia/library/SyncAppvPublishingServer.exe-3C291419F60CDF9C2E4E19AD89944FA3.html)[4 - appv](https://www.trellix.com/en-ca/about/newsroom/stories/research/suspected-darkhotel-apt-activity-update/) Proxying execution may function as a trusted/signed alternative to directly invoking `powershell.exe`.[7 - appv](https://x.com/monoxgas/status/895045566090010624)

For example,  PowerShell commands may be invoked using:[5 - appv](https://lolbas-project.github.io/lolbas/Scripts/Syncappvpublishingserver/)

`SyncAppvPublishingServer.vbs "n; {PowerShell}"`
