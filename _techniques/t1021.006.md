## Description

Adversaries may use Valid Accounts to interact with remote systems using Windows Remote Management (WinRM). The adversary may then perform actions as the logged-on user.

WinRM is the name of both a Windows service and a protocol that allows a user to interact with a remote system (e.g., run an executable, modify the Registry, modify services). [Microsoft WinRM](https://learn.microsoft.com/en-us/windows/win32/winrm/portal) It may be called with the `winrm` command or by any number of programs such as PowerShell. [Jacobsen 2014](https://www.slideshare.net/kieranjacobsen/lateral-movement-with-power-shell-2) WinRM can be used as a method of remotely interacting with Windows Management Instrumentation. [MSDN WMI](https://msdn.microsoft.com/en-us/library/aa394582.aspx)
