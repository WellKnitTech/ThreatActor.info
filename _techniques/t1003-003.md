## Description

Adversaries may attempt to access or create a copy of the Active Directory domain database in order to steal credential information, as well as obtain other information about domain members such as devices, users, and access rights. By default, the NTDS file (NTDS.dit) is located in <code>%SystemRoot%\NTDS\Ntds.dit</code> of a domain controller. [Wikipedia Active Directory](https://en.wikipedia.org/wiki/Active_Directory) 

In addition to looking for NTDS files on active Domain Controllers, adversaries may search for backups that contain the same or similar information. [Metcalf 2015](http://adsecurity.org/?p=1275) 

The following tools and techniques can be used to enumerate the NTDS file and the contents of the entire Active Directory hashes.

* Volume Shadow Copy
* secretsdump.py
* Using the in-built Windows tool, ntdsutil.exe
* Invoke-NinjaCopy
